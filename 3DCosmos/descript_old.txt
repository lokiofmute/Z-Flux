General|Store
>>>>
Stores a variable into the memory context of the program. This content will be stored as long as the program is running, and will remain available for other scripts that are running in this context.

\var{name} = the name under which it will be stored. \\
\var{content} = the content that will be stored.

Example: \\
\sourcecode{
Store("MyVar","MyContent"); \\
... \\
rs=Recall("MyVar");
} 

See also \linkitem{General|Recall}
----

General|Recall
>>>>
Recalls a variable from the memory context of the program.

See also \linkitem{General|Store}
----

General|ExecuteScript
>>>>
This function executes a script. \var{Script} provides the name of the file that contains the script (including the path and file extension). \var{Argument} provides the argument that is passed to the executed script (this script can recall this parameter using \linkitem{General|ScriptArgument})

General|CatchedError
>>>>
This function returns an error that was generated inside the last try block. If it returns an empty string, no error was generated.

Example: \\
\sourcecode{ \\
try { \\
\ \ 1/0; \\
} \\
errorstring=CatchedError;
} \\

General|createvar
>>>>
Creates a new, uninitialised variable. For most purposes, it is not necessary to call this function prior to using a variable, because the script language automatically create variables when a value is assigned to them. However, this function can be used to ensure that a variable is created at the correct scope level (e.g. at the level of the main block, and not inside a loop block).

Example: \sourcecode{createvar(i);} initialises a new variable called \sourcecode{i}.

Mathematics;Point|radial2point
>>>>
Transforms spherical coordinates $(R,\theta,\phi)$ into a point. \\
\var{R} = radius $R$ \\
\var{ang1} = $\theta$ \\
\var{ang2} = $\phi$ \\
\var{Return value} = the point. \\
The point $(x,y,z)$ will be calculated as:
\begin{equation}
\begin{array}{rcl}
x & = & R \cos \theta \cos \phi \\
y  & = & R \sin \theta \cos \phi \\
z & = & R \sin \phi
\end{array}
\end{equation}

Mathematics|Transformation
>>>>
Defines an affine transformation of the space, consisting in a translation, rotation, and anisotropic scaling.

A transformation variable can be used to map points and vectors, using the multiplication operator
(see also \linkitem{Mathematics;Transformation|Transformation*Point} and 
\linkitem{Mathematics;Transformation|Transformation*Vector})

\begin{equation} \left[
\begin{array}
a_{11} & a_{12} & a_{13} & x_t \\
a_{21} & a_{22} & a_{23} & y_t \\
a_{31} & a_{32} & a_{33} & z_t \\
0 & 0 & 0 & 1
\end{array} \right]
\end{equation}

Mathematics;Transformation|Transformation*Transformation
>>>>
Creates a new transformation that is the result of the consecutive application of two other transformations. Note that the right side transformation is applied first, followed by the left side transformation.

Example: \sourcecode{T3=T2*T1;} creates a new transformation called \sourcecode{T3} that
corresponds to the application of the transformation \sourcecode{T1}, then followed by \sourcecode{T2}.
